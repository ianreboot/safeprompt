## Implementation Details

### Critical Context

**Architecture Paradigm** (MUST UNDERSTAND):
Custom lists work as **routing signals** and **confidence modifiers**, NOT instant block/allow decisions.

**How Current System Works**:
1. Pattern Detection (XSS, SQL, etc.) ‚Üí If matched WITHOUT context ‚Üí Instant block
2. Pattern Detection WITH business keywords ‚Üí Route to AI with context ‚Üí AI decides
3. Consensus Engine ‚Üí Business confidence can override attack confidence

**How Custom Lists Work** (Same Paradigm):
1. Pattern Detection (XSS, SQL, etc.) ‚Üí If matched ‚Üí Check custom whitelist ‚Üí If NO match ‚Üí Instant block
2. If custom whitelist matches ‚Üí Route to AI with whitelist context (confidence signal 0.8)
3. If custom blacklist matches ‚Üí Route to AI with blacklist context (confidence signal 0.9)
4. Consensus Engine ‚Üí Apply same override logic using custom list confidence signals
5. **Blacklist Always Wins**: If both matched, blacklist signal (0.9) > whitelist signal (0.8)

**Key Files to Understand**:
- `/home/projects/safeprompt/api/lib/ai-validator-hardened.js` - Main validation pipeline
- `/home/projects/safeprompt/api/lib/ai-orchestrator.js` - Routing logic
- `/home/projects/safeprompt/api/lib/consensus-engine.js` - Override logic

**Things That Must Not Change**:
- Pattern detection for XSS, SQL, template injection, command injection ‚Üí Always blocks if no context
- External reference detection ‚Üí Always blocks dangerous references
- Consensus engine override thresholds (business > 0.8, attack < 0.6)
- Two-pass validation architecture

- Test #29 ("Override shipping address") passes with default whitelist containing "shipping address"

### Core Principle: Confidence Signals, Not Instant Decisions

Custom lists work as **routing signals** and **confidence modifiers** in the validation pipeline, NOT as instant block/allow gates.

### "Blacklist Always Wins" Semantics

**Confidence Levels**:
- Custom whitelist: 0.8 (business override threshold)
- Custom blacklist: 0.9 (high-confidence attack)

**Consensus Logic** (existing, no changes needed):
- Business 0.8+ overrides attack < 0.6 ‚Üí SAFE
- Attack 0.6-0.7 with business 0.8+ ‚Üí UNSAFE (borderline, flagged)
- Attack 0.75+ ‚Üí UNSAFE (always blocks)

**Result**: Blacklist (0.9) > Attack threshold (0.75) ‚Üí Always blocks, even if whitelist also matches

**Example**:
```javascript

// - business signal: 0.8 (whitelist)
// - attack signal: 0.9 (blacklist)

// Attack 0.9 > 0.75 threshold ‚Üí UNSAFE (blacklist wins)
```

### Validation Pipeline Order (Complete)

```
1. Pattern Detection (XSS, SQL, Template, Command, Semantic)
   ‚îú‚îÄ If pattern WITHOUT context ‚Üí INSTANT BLOCK
   ‚îî‚îÄ If pattern WITH custom whitelist ‚Üí route to AI with whitelist signal

2. External Reference Detection
   ‚îú‚îÄ If dangerous reference ‚Üí INSTANT BLOCK
   ‚îî‚îÄ Custom whitelist CANNOT override this

3. Custom Lists Check (NEW)
   ‚îú‚îÄ Check blacklist ‚Üí if matched, create attack signal (0.9)
   ‚îî‚îÄ Check whitelist ‚Üí if matched, create business signal (0.8)

4. AI Orchestrator
   ‚îú‚îÄ Receives: prompt + patternContext + customListContext
   ‚îî‚îÄ Routes to appropriate validators (business, attack, semantic)

5. AI Validators (parallel execution)
   ‚îú‚îÄ Business Validator: Checks for legitimate business operations
   ‚îú‚îÄ Attack Detector: Checks for manipulation attempts
   ‚îî‚îÄ Semantic Analyzer: Checks for indirect extraction

6. Consensus Engine
   ‚îú‚îÄ Aggregates validator results + custom list signals
   ‚îú‚îÄ Applies override logic (business 0.8+ vs attack < 0.6)
   ‚îî‚îÄ Returns final verdict with confidence score
```

### Security Invariants (CANNOT BE CHANGED)

**Pattern Detection Cannot Be Overridden**:
- XSS, SQL, Template, Command injection detected ‚Üí Either instant block OR route to AI
- Custom whitelist can provide context signal but NOT bypass pattern detection
- Example: Whitelist contains "script" ‚Üí `<script>alert(1)</script>` still detected as XSS
- Solution: Whitelist should use 2+ word phrases ("educational script tag example")

**External References Cannot Be Overridden**:
- Dangerous URLs, IPs, file paths ‚Üí Instant block
- Custom whitelist CANNOT allow these through
- This is for security: external references can exfiltrate data

**Consensus Override Thresholds**:
- Business override: 0.8+ (to beat attack < 0.6)
- Attack always blocks: 0.75+
- These thresholds are calibrated based on production data
